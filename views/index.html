welcome to Blue sky

The main goal of the project is to create a system that will assist in managing all processes related to the aviation traffic system in a centralized and efficient way. This includes the management of airlines companies, flights details, tickets management, and customers information.

Summary of the project

flights management system project

This project was built as the final assignment of my studies.
The project is fully responsive and works on all devices,
On the server side, the system manages all processes related to flights, airlines, customers, and tickets.
On the client side, users can register to the site, search and choose flights or airlines, and purchase or cancel tickets as needed.

I developed the entire project from scratch — both the server-side and client-side, including the full database structure , and manually entered all the data used throughout the site. On the client side, I designed and built the full user interface (UI), while on the server side, I created all the core functions and APIs needed to operate the system, including handling bugs and error responses.

There is also a separate repository available on my GitHub page, containing a version of the site that is fully set up for testing with Postman, making it easy to continue development and ensure everything works as intended.

All data is stored in a mysql database

The client side is created with

Html, Css, Javascript, bootstrap and bootstrap-icons

The server side is created with

node.js, express.js, express.router

I connected to the database using libraries

knex, mysql, mysql2

The site was built in architecture mvc- model, view, controller

Configuration

The project includes essential configuration files that serve as the foundation for the system's setup and testing.
These files include:

Task File: Describes the project tasks that was requirement, which served as a work plan for building the project.

SQL Database File: Includes the database structure, tables, relationships, and initial data (seed data) for setting up the database.

Authorization Issue Handling: This file documents the configurations and solutions for resolving access (authorization) issues in mySQL database. the file provides steps to resolve the issue, such as modifying permissions in the database or updating the server configuration file.

These files ensure that every step in the project's development, testing, and maintenance is clear, organized, and accessible for reference.
